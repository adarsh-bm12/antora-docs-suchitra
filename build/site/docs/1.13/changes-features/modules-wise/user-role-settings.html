User Role Settings

R0025

User Role settings (applicable to Nymbl Suite ERP-v0.8.3.11)

  • Update

    • A license count has been set in the Additional Info tab of Company master for User and BI user.

      While adding new users (based on the company selected), validate the user license. If the count exceeds, an error message will be shown and blocks the action of adding the user. This is applicable to both user and BI user.

      User validation

      BI User validation

    • While unfreezing the user in the edit form, validate the user license mentioned above. This will be based on companies where user has permission. This is not applicable for Default User and Device User.

    • The system will prevent POS users (users with Is Device User? enabled) from adding new roles. A validation message will be shown.

    • Disable the Is Device User? field in the new mode and edit mode.

      New mode

      Edit mode

    • The Allow BI App Login? field is disabled for the device users→ POS user will not have access to login the BI app since menu cannot be added.

    • In edit form, hide the user password visibility.

    • Need to validate the license count while adding new company permission to existing user.

R000

User Role Settings (applicable to Nymbl Suite ERP-v0.5.3.8)

Two New fields added in the user adding popup - Allow BI Access? and Local Admin (Cust)?

These two fields will be visible for only for ERP local admin where "AU_ADM_YN" field value='Y'.

  • Allow BI Access?: On enabling this field will allow the user to access the BI App.

  • Local Admin (Cust)?: On enabling this field allows the local admin user to customize the user roles and menu.(eg: new user can be added and provide permissions)
